# 研究日志 Journal

> 记录一切。没有记录就没有成长。

---

## 目录结构

```
journal/
├── README.md          # 本文件
├── ideas.md           # 策略想法记录（随时更新）
├── weekly/            # 每周复盘
│   └── YYYY-WW.md    # 如：2026-W12.md
└── strategies/        # 策略生命周期记录
    └── 策略名.md
```

---

## 想法记录格式（ideas.md）

```markdown
## YYYY-MM-DD | 想法标题

**核心逻辑：** 一句话说清楚为什么这个逻辑能赚钱

**信号定义：** 具体触发条件

**风险点：** 可能失效的原因

**状态：** 待研究 / 研究中 / 已回测 / 已放弃
```

---

## 使用原则

1. **真实记录**，包括失败的实验和错误判断
2. **数字说话**，避免模糊表述如"表现不错"
3. **标注时间**，方便回溯
4. **定期回顾**，每月重读一次，看是否有模式
