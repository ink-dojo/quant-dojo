## v22 回测结果（2026-04-16 下午）

v22 = v16 − {win_rate_60d, mom_6m_skip1m, high_52w, amihud_illiq}，
5 因子 IC 加权 + 行业中性化。

| 指标 | v16 (9因) | v22 (5因) | 差异 |
|------|-----------|-----------|------|
| Sharpe | 0.740 | **0.545** | **−0.195** |
| 年化 | 22.87% | 16.54% | −6.3 pts |
| 最大回撤 | −43.06% | −40.86% | +2.2 pts |
| DSR | 90.7% | 84.4% | −6.3 pts |
| 候选池排名 | **#1** | #8 / 14 | ↓ |

**结论：剪枝不但没帮上，反而伤了 Sharpe。**

教训:
1. **"Gram-Schmidt 残差 IC 95%CI 跨零" ≠ "因子是噪声"**。
   残差 IC 是"边际线性信息"的统计显著性；但在 IC-加权组合里，弱但
   正向的因子仍然通过加权平均贡献组合信号。残差 CI 的窄幅 ≠ 零贡献。
2. 9 因子 → 5 因子后 IC 加权权重集中在少数强因子，失去了 regime
   分散（动量、反转、波动 三类因子的互补性）。
3. **候选池 σ 窄的真正原因**可能不是因子冗余，而是**候选池构造方式**：
   所有 v11-v21 都共享 v13 骨架（team_coin, low_vol, high_52w,
   mom_6m_skip1m），变动幅度小，自然同质。

诚实取舍：保留 v16 为 candidate face；转去解决真正的瓶颈——最大回撤。
`mdd=−43%` 才是所有候选通不过 admission gate 的根本原因。

## v23 回测结果（2026-04-16 晚）

v23 = v16 + adaptive_half_position_stop（baseline=-0.10, vol_window=60, ref_vol=0.15）

参数选自 `sweep_v16_dd_overlay.py` 的 IS 扫描前 2 名（14 组）。

| 指标 | v16 | v23 (IS 扫描预测) | v23 (fresh 回测) |
|------|-----|-------------------|------------------|
| Sharpe | 0.740 | 0.837 | **0.674** |
| 年化 | 22.87% | 18.21% | **14.81%** |
| 最大回撤 | −43.06% | −26.42% | −28.18% |
| DSR 排名 | #1 (90.60%) | — | #4 (88.37%) |
| Gate ann≥15% | ✗（过）✓ | ✓ | ✗（差 0.19pp） |
| Gate sharpe≥0.8 | ✗ | ✓ | ✗ |
| Gate mdd>-30% | ✗ | ✓ | **✓** |
| Gate all pass | ✗ | ✓ | ✗ |

**结果：IS 扫描预测失真**。fresh 回测 sharpe 从 0.84 降到 0.67，
年化从 18.2% 掉到 14.8%。MDD 确实压到 -30% 以内（-28.18%），但代价
比扫描预测重得多。

教训:
1. IS 扫描（14 组参数）挑出的 "最佳" 参数在同一条 equity 上测，
   本质是 in-sample 过拟合。即便 sweep 的 equity 来自真实历史 v16
   回测，挑最高 sharpe 的参数组仍吸收了噪声波动。
2. 自适应半仓止损确实有效压 MDD，但 A 股高波动期的 "假回调" 仍会
   在它刚触发后上涨，错过 reentry（半仓需净值创新高才恢复）。
3. 真正的解决方案很可能是：
   - 先把 stop 参数用 WF 学（分 2 期滚动），而不是 IS 挑最高
   - 或更激进的仓位管理（vol targeting + regime filter）而非单纯 stop
   - 或从因子层面降 MDD：加入对冲/空头因子或行业中性化加强

诚实取舍：v23 作为候选保留以记录这条路径，但不是 best-in-pool。
v16 仍是 IS sharpe 最高，但 MDD -43% 无法上线。**15 个候选全部
未过 admission gate**，下一步需转向 WF 验证而非继续加候选。
