# 工作流规范 WORKFLOW

> 专业不是天赋，是习惯。这份文档定义我们如何工作。

---

## 策略生命周期

每个策略必须经历以下阶段，**不能跳过**：

```
想法 → 假设 → 研究 → 回测 → 评审 → 模拟 → 实盘
 │       │       │       │       │       │       │
记录   记录    notebook  报告   PR review  日志   日志
```

### 1. 想法阶段
- 在 `journal/ideas.md` 里用一句话记录
- 写清楚：**为什么这个逻辑能赚钱？经济学解释是什么？**
- 还没有代码，只有思考

### 2. 假设阶段
- 把想法转化为可验证的假设
- 例："过去20天涨幅排名前20%的股票，未来5天会继续跑赢市场"
- 定义清楚：信号、持仓期、预期收益来源

### 3. 研究阶段
- 在 `research/notebooks/` 下创建新 notebook
- 命名规范：`YYYYMMDD_策略名_研究内容.ipynb`
- 先做探索性分析，验证假设是否有统计支撑
- **不要在这里就开始优化参数**

### 4. 回测阶段
- 研究通过后，在 `strategies/` 下实现策略代码
- 必须包含：样本内 + 样本外测试
- 必须计算：夏普比率、最大回撤、年化收益、胜率、盈亏比
- 必须考虑：手续费（双边0.3%）、滑点（1个tick）

### 5. 评审阶段
- 提 Pull Request，对方 review
- Checklist（见下方）
- 讨论完毕后合并

### 6. 模拟阶段
- 通过评审的策略进入模拟盘
- 每周在 `journal/` 记录实际表现 vs 回测预期
- 模拟3个月以上，行情需覆盖上涨/下跌/震荡

### 7. 实盘阶段
- 模拟验证通过后，投入真实资金
- 从小仓位开始（建议不超过总资金10%）

## Git 工作流

### 分支命名
```
main          # 稳定版本，只接受 PR 合并
dev           # 开发主分支
feature/xxx   # 新功能 / 新策略
research/xxx  # 研究性探索（可以随意，但不进 main）
fix/xxx       # Bug 修复
```

### 提交规范
```
feat: 添加双均线策略
fix: 修复回测引擎的未来函数 bug
research: 动量因子 IC 分析
docs: 更新 ROADMAP 第二阶段
refactor: 重构 metrics 模块
```

### PR 流程
1. 从 `dev` 切出 `feature/` 分支
2. 完成开发，本地测试通过
3. 提 PR 到 `dev`，写清楚：做了什么、为什么、怎么测试的
4. 对方 review，必须至少1个 approval
5. 合并后删除 feature 分支
6. 重要版本从 `dev` 合并到 `main`

## 策略评审 Checklist

提 PR 前自检，review 时对方检查：

**逻辑合理性**
- [ ] 有明确的经济学解释（不是纯数据挖掘）
- [ ] 信号的因果逻辑说得通

**回测质量**
- [ ] 没有未来函数（look-ahead bias）
- [ ] 没有幸存者偏差（使用当时可用的股票池）
- [ ] 交易成本合理计入
- [ ] 样本外（out-of-sample）测试通过

**绩效标准（最低门槛）**
- [ ] 年化收益 > 15%
- [ ] 夏普比率 > 0.8
- [ ] 最大回撤 < 30%
- [ ] 回测时间跨度 > 3年，覆盖牛熊周期

**代码质量**
- [ ] 代码可读，有必要注释
- [ ] 函数职责单一
- [ ] 无硬编码参数（参数在 config 里）

| 时间 | 内容 |
|------|------|
| 周一 | 上周复盘，本周计划 |
| 周中 | 各自推进研究/开发 |
| 周末 | 同步进展，互相 review，更新 journal |

## 数据规范

- 原始数据放 `data/raw/`，**不入 git**（.gitignore 已配置）
- 处理后数据放 `data/processed/`，**不入 git**
- 数据获取脚本放 `utils/data_loader.py`，**入 git**
- 所有数据使用前必须检查：缺失值、异常值、复权处理

---

## 风控底线（任何时候不得违反）

1. **单策略最大仓位不超过总资金 30%**
2. **组合最大回撤触达 15% 时，强制减仓至半仓**
3. **最大回撤触达 20% 时，全部清仓，复盘后再入场**
4. **任何策略实盘前必须有模拟盘记录**
5. **永远不加杠杆，直到系统稳定盈利1年以上**
